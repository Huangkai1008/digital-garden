# 第6章 存储

存储是数据工程生命周期的基石（图 6-1），支撑其主要阶段——摄取、转换和服务。数据在生命周期中流动时会被多次存储。无论数据是秒、分钟、天、月还是年后需要，它都必须持久化在存储中，直到系统准备好消费它进行进一步处理和传输。了解数据的用例和未来检索方式是选择合适存储解决方案的第一步。

## 存储的原始成分

理解物理存储技术的基本特征对于评估任何存储架构固有的权衡至关重要。

### 磁盘驱动器（HDD）

**磁盘驱动器**利用涂有铁磁膜的旋转盘片。每 GB 成本约 3 美分，是批量数据存储的骨干。主要限制：寻道时间、旋转延迟、IOPS（50–500）。

### 固态硬盘（SSD）

**SSD**在闪存单元中以电荷存储数据，消除机械组件。随机查找 <0.1 ms，支持数千 IOPS。每 GB 约 20–30 美分，是 OLTP 系统的标准。

### 随机存取存储器（RAM）

**RAM**易失、高速（约 100 ns 延迟）、昂贵（约 $10/GB）。用于缓存、数据处理、索引。

### 网络与 CPU

分布式存储系统依赖网络和 CPU 聚合读取、分发写入。可用区（availability zone）是标准云构造。

### 序列化、压缩、缓存

**序列化**将数据打包成可解码的标准格式。**压缩**减小数据体积，提高有效扫描速度。**缓存**将频繁或最近访问的数据存储在快速访问层。

## 数据存储系统

| 类型 | 描述 |
|------|------|
| 文件存储 | 层次化文件系统，NFS、S3 兼容 |
| 块存储 | 原始块设备，用于数据库 |
| 对象存储 | 键值存储，S3、GCS、Azure Blob |
| 缓存与内存 | Redis、Memcached |
| HDFS | Hadoop 分布式文件系统 |
| 流式存储 | Kafka、Pulsar |

## 数据工程存储抽象

- **数据仓库**：面向分析的结构化存储
- **数据湖**：原始和半结构化数据
- **数据湖屋**：结合两者
- **数据平台**：集成存储、计算、服务
- **流到批存储架构**：流式数据落地为批处理

## 存储大趋势

- 数据目录（data catalog）
- 数据共享（data sharing）
- 模式（schema）管理
- 计算与存储分离
- 数据生命周期与保留
- 单租户与多租户

## 小结

存储贯穿整个数据工程生命周期。理解原始成分（HDD、SSD、RAM、网络）、存储系统（文件、块、对象、流式）和抽象（仓库、湖、湖屋）的权衡，是设计稳健数据架构的基础。
