# 第8章 查询、建模与转换

到目前为止，数据工程生命周期的阶段 primarily 是关于将数据从一处传递到另一处或存储它。本章你将学习如何使数据有用。通过理解查询、建模和转换（图 8-1），你将掌握将原始数据转化为下游利益相关者可消费产物的工具。

## 查询

**查询（query）**允许你检索并对数据执行操作。查询是 CRUD 中的 R（读取）。

### SQL 语言类别

- **数据定义语言（DDL）**：CREATE、DROP、ALTER——定义数据库对象状态
- **数据操作语言（DML）**：SELECT、INSERT、UPDATE、DELETE、COPY、MERGE——添加和修改数据
- **数据控制语言（DCL）**：GRANT、DENY、REVOKE——控制访问
- **事务控制语言（TCL）**：COMMIT、ROLLBACK——控制事务

### 查询的生命周期

1. 数据库引擎编译 SQL，解析代码检查语义
2. SQL 转换为字节码
3. 查询优化器分析字节码，确定执行计划
4. 执行查询，产生结果

### 提升查询性能

- **优化 join 策略和模式**：预 join、简化 join 条件、使用 CTE
- **避免行爆炸（row explosion）**：注意多对多匹配
- **使用 EXPLAIN 计划**：理解优化器决策
- **避免全表扫描**：使用剪枝、分区、聚类
- **利用物化视图**：预计算常用查询

## 数据建模

**数据建模（data modeling）**是将数据转换为可用形式的过程。常见模式包括：

- **Kimball**：星型/雪花模式，面向维度建模
- **Inmon**：企业信息工厂，3NF 规范化
- **Data Vault**：中心-链接-卫星，面向历史审计

## 转换

**转换（transformation）**将数据从原始形式变为对下游用例有用的形式。转换类型包括：

- 类型映射与格式标准化
- 模式转换与规范化
- 聚合与报表
- ML 特征工程（featurization）

转换可在批处理或流式（in-flight）中执行。业务逻辑是数据转换的主要驱动力。

## dbt 与管道即代码

dbt（Data Build Tool）等工具将数据建模和转换作为代码管理，支持版本控制、测试和文档化。

## 小结

查询、建模和转换是将原始数据转化为下游价值的核心活动。理解查询优化、数据建模模式、转换逻辑以及 dbt 等现代工具，是数据工程师的关键技能。
