# 第9章 数据服务：分析、ML 与反向 ETL

恭喜！你已到达数据工程生命周期的最后阶段——为下游用例提供数据服务（图 9-1）。本章你将了解作为数据工程师将遇到的三大用例的数据服务方式：分析/BI、机器学习、反向 ETL。

## 数据服务的一般考量

### 信任

> 建立声誉需要 20 年，毁掉它只需 5 分钟。
>
> — Warren Buffett

信任是数据服务的根本考量。最终用户需要信任他们接收的数据。通过数据验证、数据可观测性、SLA/SLO 建立信任。

### 用例与用户

数据的最佳用途是促成行动。问自己：这些数据将触发什么行动？谁将执行？能否自动化？优先高 ROI 用例。

### 数据产品

> 数据产品是通过使用数据促进最终目标的产品。
>
> — D. J. Patil

考虑「待完成工作（jobs to be done）」——用户「雇用」产品完成「工作」。数据产品应有正向反馈循环。

### 自助服务与否

自助服务数据产品是多年夙愿，但实践中很难。成功关键在于找到合适的受众——有数据背景的高管、愿意学习数据技能的业务领导。

### 数据定义与逻辑

**数据定义**指组织内对数据含义的理解。**数据逻辑**规定从数据推导指标的公式。在数据目录和生命周期各阶段正式声明定义与逻辑。

### 数据网格

**数据网格（data mesh）**改变组织内数据服务方式：每个域团队负责向其他团队提供数据，并可能运行自己的仪表板和分析。

## 服务分析数据

- **商业智能（BI）**：描述业务过去和当前状态
- **运营分析**：细粒度运营细节，可立即行动
- **嵌入式分析**：面向客户的分析，多租户、高并发

## 服务 ML 数据

- 高质量、适当准备的数据是 ML 的前提
- **特征存储（feature store）**：维护特征历史和版本，支持团队间共享
- 数据工程师与数据科学家、ML 工程师协作

## 反向 ETL

**反向 ETL**将处理后的数据从数据工程生命周期输出端反馈回源系统。例如：从广告平台获取数据、计算每次点击出价、将出价反馈回广告平台。Hightouch、Census 等工具支持反向 ETL。

## 小结

数据服务是生命周期的最后阶段，将数据价值交付给分析、ML 和运营系统。信任、用例理解、数据产品思维、SLA/SLO 是成功的关键。暗流贯穿服务阶段。
