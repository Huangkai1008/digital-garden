# 第10章 大规模数据处理的演进

欢迎来到本书最后一章。为收尾，我们将简要回顾历史，从 MapReduce 的「远古」时代开始，触及随后十五年中的一些亮点，这些将流式系统带到了今天的位置。

本章聚焦 MapReduce/Hadoop 系大规模数据处理的特定脉络，不涉及 SQL 领域、HPC/超级计算机等。同时会较多涉及 Google 的技术，既因作者背景，也因大数据对 Google 一直重要，且外界对 Google 内部工作往往有了解兴趣。

---

## MapReduce（2003）

大规模数据处理可以说始于 2003 年的 MapReduce。当时 Google 工程师为应对 Web 规模的数据处理挑战构建各种定制系统，注意到三件事：数据处理难、可扩展性难、容错难。他们得出结论：若能构建一个解决后两者（可扩展性和容错）的框架，专注前者会简单得多。MapReduce 由此诞生。

MapReduce 的基本思想是提供以函数式编程中两个熟知操作（map 和 reduce）为中心的简单数据处理 API，管道在负责可扩展性和容错的分布式框架上执行。如第 6 章所述，Map 和 Reduce 阶段在高层都做三件事：将表转为流、对流应用用户变换产生新流、将流分组为表。

MapReduce 论文发表于 OSDI 2004，但未提供源码。此后十年 MapReduce 在 Google 内持续发展，规模达到前所未有的水平。从流式系统视角，MapReduce 的主要启示是**简洁性**和**可扩展性**。

---

## Hadoop（2005）

2005 年，Doug Cutting 和 Mike Cafarella 在构建 Nutch 爬虫的分布式版本时，决定采用 MapReduce 论文的思想。他们已构建了 Google 分布式文件系统的自己的版本（NDFS，后更名 HDFS），自然下一步是在论文发表后添加 MapReduce 层。他们称该层为 Hadoop。

Hadoop 与 MapReduce 的关键区别是：Cutting 和 Cafarella 将 Hadoop（及 HDFS）的源码开源，成为 Apache Hadoop 项目的一部分。Yahoo 聘请 Cutting 帮助将爬虫架构迁移到 Hadoop，为项目带来额外认可和工程投入，此后整个开源数据处理工具生态蓬勃发展。

本节的主要启示是：围绕 Hadoop 开花的**开源生态**对行业整体产生了巨大影响。通过创建开放社区，让工程师改进和扩展早期 GFS 和 MapReduce 论文的思想，诞生了繁荣的生态，产生了 Pig、Hive、HBase、Crunch 等众多工具。

---

## Flume（2007）

Flume（又称 FlumeJava，指其 Java 版本；勿与 Apache Flume 混淆）是 Google 内 MapReduce 的正式继任者，由 Craig Chambers 在 2007 年西雅图办公室成立时创立。

动机是解决 MapReduce 在成功几年后显现的固有缺陷，许多围绕其僵化的 Map → Shuffle → Reduce 结构：

- 许多用例无法由单次 MapReduce 满足，出现了各种定制编排系统协调 MapReduce 作业序列，它们互不兼容
- 清晰的 MapReduce 作业序列常因 API 的僵化结构引入低效
- 为优化这些低效，工程师引入手动优化，使管道逻辑变得晦涩

Flume 通过提供可组合的高层 API（基于与 Beam 相同的 PCollection 和 PTransform 概念）描述数据处理管道，解决了这些问题。管道启动后会经过**优化器**生成最优 MapReduce 作业序列的执行计划。

重要优化包括：**融合（fusion）**——将逻辑上独立的阶段在同一作业中顺序或并行运行；**Combiner 提升**——在 group-by-key 之前进行部分聚合，减少网络 shuffle 量。Flume Java 于 2009 年初在 Google 推出后大获成功，后迁移到自定义执行引擎 Dax，不再使用 MapReduce。

---

## MillWheel、Kafka、Flink、Spark、Cloud Dataflow

- **MillWheel**：Google 的原始通用流处理架构，提供低延迟、强一致处理，是 Cloud Dataflow 的前身之一。
- **Kafka**：持久化流式传输层，实现为分区日志集。Kafka 在推动流式处理方面发挥了最具影响力的作用之一，其「流与表」思想与本书理论一致。
- **Flink**：原生流式引擎，底层全流式，即使在「批处理」模式下也始终流式。
- **Spark**：通过 Spark Streaming（微批）和 Structured Streaming 支持流式。
- **Cloud Dataflow**：统一批处理+流式编程模型，融合了本书讨论的变换、窗口化、水位线、触发器、累积等概念。

---

## 小结

大规模数据处理从 MapReduce 的简洁与可扩展性起步，经 Hadoop 的开源生态蓬勃发展，由 Flume 等系统解决编排与优化问题，再由 MillWheel、Kafka、Flink、Spark、Dataflow 等将流式处理推向成熟。这些贡献共同塑造了今天的现代流式处理生态。
