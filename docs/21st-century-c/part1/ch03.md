如果你读到这里，你已经接触了解决 C 代码核心问题的工具，如调试和文档化。如果你急于进入 C 代码本身，可以跳到 Part II。本章和下一章将涵盖面向协作和分发的重量级工具：包构建工具和版本控制系统。

C 社区对互操作性有很高标准。如今，Autotools——为给定系统自动生成完美 makefile 的系统——是代码分发的核心。你已在「从源代码使用库」中用过它来快速安装 GNU Scientific Library。

但除非你对 makefile 的工作原理有清晰认识，否则很难跟上 Autotools 在做什么，所以我们需要先更详细地介绍这些。粗略地说，makefile 是有组织的 shell 命令集合，因此你需要了解 shell 为自动化工作提供的各种设施。路径很长，但最终你将能够：

- 使用 shell 自动化工作
- 使用 makefile 组织 shell 执行的所有任务
- 使用 Autotools 让用户在任何系统上自动生成 makefile

## Shell

POSIX 标准 shell 具有：丰富的宏设施（展开语法）、图灵完备的编程语言、交互前端（命令提示符）、记录和重用输入的历史系统等。

反引号 `` ` `` 用命令的输出替换命令，以宏风格进行。示例 3-1 展示用 shell 变量和 POSIX 工具统计 C 代码行数的脚本：

**示例 3-1. 用 shell 变量和 POSIX 工具统计行数（linecount.sh）**

```bash
Lines=`grep '[;)}]' *.c | wc -l`
Files=`ls *.c |wc -l`
echo files=$Files and lines=$Lines
echo lines/file = $(($Lines/$Files))
bc << ---
scale=3
$Lines/$Files
---
```

用 `for` 循环操作文件集：

```bash
for file in *.c; do cp $file ${file}.bkup; done
```

## 测试文件是否存在

使用 `test` 命令检查文件是否存在：

```bash
test -e a_file_i_know
echo $?
```

`[` 和 `]` 等价于 test。示例 3-3 在 if 语句中使用 test：

**示例 3-3. 围绕 test 的 if/then 语句（iftest.sh）**

```bash
if [ ! -e a_test_file ]; then
    echo test file had not existed
    touch a_test_file
else
    echo test file existed
    rm a_test_file
fi
```

## Makefile 与 Shell 脚本

Makefile 的每一行在单独 shell 中运行。若要在 makefile 中跨行共享变量，使用反斜杠续行，且每行以单个反斜杠结尾（不能有尾随空格）。

## 用 Autotools 打包代码

Autotools 由三个主要工具组成：

1. **Autoconf**：生成 configure 脚本，检测系统特性
2. **Automake**：从 Makefile.am 生成 Makefile
3. **Libtool**：管理共享库的编译和链接

典型工作流：

```bash
autoreconf -iv
./configure
make distcheck
```

**Makefile.am 示例：**

```makefile
bin_PROGRAMS = weather wxpredict
weather_SOURCES = weather.c
wxpredict_SOURCES = wxpredict.c
AM_CFLAGS = $(OPENMP_CFLAGS) -g -Wall -O3
weather_LDADD = -lgsl -lgslcblas -lm
wxpredict_LDADD = -lgsl -lgslcblas -lm
```

**configure.ac 基本结构：**

```makefile
AC_INIT([my_package], [1.0])
AM_INIT_AUTOMAKE
AC_PROG_CC
AC_CONFIG_FILES([Makefile])
AC_OUTPUT
```

## VPATH 构建

可以从任意目录编译，不必在源码目录中：

```bash
cd ~/build
~/pkgsrc/configure
make
```
