# 第5章 与他人协作

编程语言的数量趋近无穷，其中很大一部分有 C 接口。本章简要介绍流程，并详细演示与一种语言——Python——的接口。

每种语言都有自己的打包和分发习惯，意味着在 C 和宿主语言中写好桥接代码后，还要面对让打包系统编译和链接一切的任务。

## 动态加载（Dynamic Loading）

在跳入其他语言之前，值得花点时间了解使一切成为可能的 C 函数：`dlopen` 和 `dlsym`。这些函数打开动态库并从库中提取符号（如静态对象或函数）。

「共享对象文件」这个名字很贴切：这种文件包含供其他程序使用的对象列表，包括函数和静态定义的结构体。

使用共享对象文件很像从保存项目列表的文本文件中检索项目。对文本文件，你先调用 `fopen` 获取文件句柄，然后调用适当函数搜索文件并返回指向找到项目的指针。对共享对象文件，打开文件的函数是 `dlopen`，搜索所需符号的函数是 `dlsym`。魔法在于你能用返回的指针做什么。若用 `dlsym` 获取函数指针，你可以调用该函数；若获取结构体指针，可以立即将结构体用作已初始化的对象。

**示例 5-1. 向用户请求函数、现场编译并运行的程序（dynamic.c）**

```c
#include <dlfcn.h>
#include <stdio.h>
#include <stdlib.h>
#include <readline/readline.h>
void get_a_function(){
    FILE *f = fopen("fn.c", "w");
    fprintf(f, "#include <math.h>\n"
               "double fn(double in){\n");
    char *a_line = NULL;
    char *prompt = ">>double fn(double in){\n>> ";
    do {
        free(a_line);
        a_line = readline(prompt);
        fprintf(f, "%s\n", a_line);
        prompt = ">> ";
    } while (strcmp(a_line, "}"));
    fclose(f);
}
void compile_and_run(){
    if (system("c99 -fPIC -shared fn.c -o fn.so")!=0){
        printf("Compilation error.");
        return;
    }
    void *handle = dlopen("fn.so", RTLD_LAZY);
    if (!handle) printf("Failed to load fn.so: %s\n", dlerror());
    typedef double (*fn_type)(double);
    fn_type f = dlsym(handle, "fn");
    printf("f(1) = %g\n", f(1));
    printf("f(2) = %g\n", f(2));
    printf("f(10) = %g\n", f(10));
}
int main(){
    printf("I am about to run a function. But first, you have to write it for me.\n"
        "Enter the function body. Conclude with a '}' alone on a line. \n\n");
    get_a_function();
    compile_and_run();
}
```

## 动态加载的局限

`dlsym` 只能从共享对象库中提取源文件中声明为文件全局的静态变量或函数。局部变量不行。即使我们只把 C 语言视为函数和全局变量，仍有广泛可能性：函数可以按需创建新对象，全局变量可以是持有函数列表的结构体。

## 编写供非 C 调用的代码

- **宏**：预处理器读取宏，最终共享库中无迹可寻。从 C 外部链接时不会有这些宏，包装函数必须复刻函数调用宏所做的。
- **接口函数数量**：每个通过 dlsym 获取的对象都需要在宿主侧提供额外冗余工作，因此限制接口函数数量至关重要。
- **对象**：结构体加若干接口函数（struct_new、struct_copy、struct_free 等）很适合这种场景。

## 包装函数

对每个期望用户调用的 C 函数，宿主侧也需要包装函数。用途包括：客户服务（宿主语言用户不必考虑 C 调用系统）、进出转换（宿主与 C 数据类型间的转换）。

## 跨境传递数据结构

将 C 结构体传给宿主语言时，通常传递不透明指针（opaque pointer）。宿主将其作为句柄保存，在调用 C 函数时传回。C 端负责分配和释放，宿主端只需持有指针。

## 编译与链接
