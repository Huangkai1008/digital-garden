# 第8章 教科书常常不讲的重要语法

上一章讲了传统 C 教科书强调但当前计算环境可能无关的主题。本章讲许多教科书不涉及或仅顺带提及的内容，分为三大部分：

- **预处理器**：宏能做的事 C 语言其余部分做不到；预处理器也是我们检测和响应环境特性的方式
- **链接与 static/extern**：不少教科书甚至不提及 static 和 extern
- **const**：太有用不能不用，但标准中的规定和常见编译器实现有些古怪

## 培养健壮繁茂的宏

有人说宏本身容易出错应避免，但那些人不会建议你不用 NULL、isalpha、assert 或标准库中通过宏定义的数十种设施。那些是写得好、每次都能正确工作的健壮宏。

宏执行文本替换（称为展开），需要与普通函数不同的思维方式，因为输入文本可能与宏内文本及源代码中其他文本交互。

**三条规则：**

1. **括号！** 宏粘贴文本时容易破坏预期。`#define double(x) 2*x` 在 `double(1+1)*8` 时展开为 `2*1+1*8` 得 10 而非 32。应写 `#define double(x) (2*(x))`。表达式型宏把整个展开式包在括号里。

2. **避免双重求值**。`#define max(a,b) ((a)>(b)?(a):(b))` 在 `max(x, y++)` 时会让 y++ 求值两次。块型宏可在块开头声明变量保存输入，然后使用副本。

3. **块用花括号**。块型宏用花括号包裹，避免与 if 等结合时出错，也避免内部声明的变量与外层冲突。

## 预处理器

`#` 有三种用途：标记指令、字符串化输入、连接 token。

**字符串化：** `#x` 将 x 转为字符串字面量。

**测试宏：** 用 `#ifdef`、`#if defined()` 等根据环境特性条件编译。

## static 与 extern

- **static** 在文件作用域：内部链接，仅本文件可见
- **static** 在函数内：静态存储期，仅本函数可见
- **extern**：声明在其他地方定义的变量，用于跨文件共享

## const

读声明的技巧：从右向左读。`const int *p` 表示 p 是指向 const int 的指针；`int * const p` 表示 p 是 const 的指向 int 的指针。

名词-形容词形式：`const` 修饰其左边的类型。
