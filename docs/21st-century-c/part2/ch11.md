典型库的常见格式（C 或任何语言）：

- 表示库所涉领域关键方面的一小组数据结构
- 操作这些数据结构的函数集（常称接口函数）

这是组织程序或库的明智方式。作者可用与问题相符的名词和动词表示概念。

对 OOP 的几种常见扩展：继承、虚函数、作用域精细控制、运算符重载、引用计数。本章考虑如何在 C 中实现这些。都不特别难：引用计数基本只需维护计数器；函数重载用 `_Generic`；虚函数可通过分派函数实现，可选地由替代函数的键值表支持。

## 扩展结构体与字典

基本形式：结构体加操作该结构体的函数集。扩展方式：如何向结构体添加新元素，如何添加对既有和新结构体都正确工作的新函数。

## 继承

通过将子结构体的第一个成员设为父结构体类型，可实现继承。子结构体指针可安全转换为父结构体指针。

## 虚函数与 vtable

在结构体中放函数指针表（vtable），不同实例可指向不同实现，实现多态。

## _Generic

C11 的 `_Generic` 根据表达式类型选择不同实现，实现函数重载风格接口。

## 封装

用 static 函数和文件作用域实现私有性。将「私有」成员放在结构体末尾，通过不公开其偏移来约定不直接访问。

## 引用计数

在结构体中维护引用计数，`_ref` 和 `_unref` 函数增加/减少计数，计数归零时释放。

核心观点：struct 加配套函数仍是世界上大量 C 库的主力格式。C 让我们能实现多种扩展，但从不强迫，在收益不大时可省略。
