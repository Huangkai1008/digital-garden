# 第12章 并行线程

> It's 99 revolutions tonight.
> —Green Day, "99 Revolutions"

近年来销售的电脑——甚至许多手机——几乎都是多核的。单线程程序无法充分利用硬件资源。幸运的是，让程序变成并发并行线程通常只需一行额外代码。

本章涵盖：

- 编写并发 C 代码的若干标准与规范概览
- 让 for 循环多线程化的一行 OpenMP 代码
- 编译 OpenMP 或 pthreads 所需的编译器标志
- 何时安全使用该魔法行的考量
- 实现 map-reduce
- 并行运行若干独立任务的语法
- `_Thread_local` 关键字
- 临界区与互斥量
- OpenMP 原子变量
- POSIX 线程及其与 OpenMP 的区别

## 环境

直到 2011 年 12 月修订，线程机制才成为标准 C 的一部分。可选方案：

- **POSIX 线程（pthreads）**：约 1995 年定义
- **Windows 线程**：CreateThread 等
- **OpenMP**：用 `#pragma` 和少量库函数告诉编译器何时如何线程化，一行代码即可将串行 for 循环变为线程化循环
- **C 标准库**：现包含线程和原子变量操作的头部

OpenMP 更易写、更少 bug，几乎所有主要编译器支持。

## 一行并行化

```c
#pragma omp parallel for
for (int i=0; i<n; i++){
    // 循环体
}
```

编译时加 `-fopenmp`（gcc）或等效标志。

## 临界区与互斥量

不可并行化的代码段标记为临界区。OpenMP 用 `#pragma omp critical`；pthreads 用 `pthread_mutex_t`。

## 线程局部变量

`_Thread_local` 为全局静态变量创建线程私有副本。

## 原子变量

OpenMP 提供原子操作；C11 的 `_Atomic` 和 `stdatomic.h` 提供标准原子变量。

## Map-Reduce

OpenMP 的 `reduction` 子句可简化归约模式。
