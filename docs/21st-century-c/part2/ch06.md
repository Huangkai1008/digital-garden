# 第6章 你的好朋友——指针

> He's the one who likes all our pretty songs / And he likes to sing along / And he likes to shoot his gun / But he don't know what it means.
> —Nirvana, "In Bloom"

像关于音乐的歌或关于好莱坞的电影一样，指针是描述其他数据的数据。很容易被淹没：你同时要应对在引用的引用中迷失、别名、内存管理和 malloc。但我们可以把这些问题拆成独立组件。例如，我们可以把指针当别名用而不碰 malloc，malloc 不必像 90 年代教科书说的那样频繁出现。

本章主题针对常见错误和常见困惑点。若你写 C 已久，这些会像第二天性，可以跳过或快速浏览。

## 自动、静态和手动内存

C 提供三种基本内存管理模型，比大多数语言多两种，也比你想关心的多两种。此外还有线程局部内存和 mmap 内存。

| 类型 | 说明 |
|------|------|
| **自动（Automatic）** | 首次使用时声明，离开作用域时移除。函数内无 static 的变量都是自动的。 |
| **静态（Static）** | 在程序整个生命周期存在于同一位置。函数外或带 static 的变量。未初始化则置零。 |
| **手动（Manual）** | 涉及 malloc 和 free，是大多数段错误的来源。唯一能在声明后调整数组大小的类型。 |

## 栈与堆

每个函数在内存中有一块空间——帧（frame），保存函数信息，如返回位置和所有自动分配变量的空间。函数调用时，新函数的帧被压入帧栈；函数完成时，其帧弹出，该帧中所有变量随之消失。

栈有任意大小限制，通常约 2–3 MB。通过 malloc 分配的内存不在栈上，而在堆（heap）中。堆的大小通常约等于可用内存。

## 数组与指针

`int an_array[32]` 与 `int *a_pointer` 的区别：

- `an_array`：在栈上预留 32 个整数的空间，an_array 是指向该空间的指针，且不能重新赋值指向别处。空间是自动分配的，不能调整大小。
- `a_pointer`：只声明指针，未绑定到任何位置，可自由赋值，如 `a_pointer = malloc(32*sizeof(int))` 或 `a_pointer = an_array`。

在函数参数中，`int *a_pointer` 和 `int an_array[]` 行为相同，因为不分配内存。

## 指针算术

你只需知道的指针算术：`ptr + 1` 指向下一个元素（不是下一个字节）。对 `int *p`，`p+1` 前进 `sizeof(int)` 字节。

## 结构体复制

对结构体，等号复制内容（浅拷贝）。若结构体含指针，只复制指针值，不复制所指数据。对数组，等号复制别名，不复制数据本身。深拷贝需要遍历指针并复制所指内容。

## 声明解读

解读声明的技巧：从右向左读。`int *p` 表示 p 是指向 int 的指针。`*` 在声明行表示指针，在非声明行表示解引用。
