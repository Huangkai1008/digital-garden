# 第9章 更简单的文本处理

> I believe that in the end the word will break cement.
> —Pussy Riot,  paraphrasing Aleksandr Solzhenitsyn

字母串是长度不定的数组，而自动分配的数组（在栈上）不能调整大小，这就是 C 中文本问题的 nutshell。幸运的是，许多前人已面对此问题并给出至少部分解决方案。少量 C 标准和 POSIX 标准函数足以满足我们许多字符串构建需求。

C 设计于 1970 年代，早于非英语语言的发明。有了正确的函数（和对语言编码的正确理解），C 最初对英语的关注不是真正问题。

## 用 asprintf 减轻字符串处理痛苦

`asprintf` 分配你所需的字符串空间并填充，意味着你几乎不必再担心字符串分配。

asprintf 不是 C 标准的一部分，但在有 GNU 或 BSD 标准库的系统上可用。GNU Libiberty 也提供可裁剪粘贴或通过 `-liberty` 链接的版本。

**示例 9-2. 使用 asprintf 的版本（getstrings.c）**

```c
#define _GNU_SOURCE
#include <stdio.h>
#include <stdlib.h>
void get_strings(char const *in){
    char *cmd;
    asprintf(&cmd, "strings %s", in);
    if (system(cmd)) fprintf(stderr, "something went wrong running %s.\n", cmd); 
    free(cmd);
}
int main(int argc, char **argv){
    get_strings(argv[0]);
}
```

若 asprintf 不可用，可用 `vsnprintf` 两次：第一次用 1 字节缓冲区获取所需长度，再分配并真正写入。

## Sasprintf 宏

将 asprintf 包装成宏可简化使用，并处理 `free` 的职责。

## strdup 与 strndup

`strdup` 分配并复制字符串；`strndup` 限制复制的最大长度。两者都是 POSIX 标准。

## 常量字符串

字符串字面量（如 `"hello"`）存储在只读内存。修改它们会段错误。用 `strdup` 得到可修改副本。

## Unicode

C 的 `char` 是字节，不是字符。UTF-8 用多字节编码字符。许多标准库函数对 UTF-8 安全：`strdup`、`strndup`、`strlen`（返回字节数）等。对完整 Unicode 支持，可用 GLib 的 `g_utf8_*` 函数或 `libunistring`。

## 国际化（i18n）

用 `gettext` 进行字符串翻译。将代码中的 `"Human message"` 替换为 `_("Human message")`。`_` 是宏，最终展开为根据用户运行时 locale 选择正确字符串的函数调用。
