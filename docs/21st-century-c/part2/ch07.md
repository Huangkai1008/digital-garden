# 第7章 教科书花大量篇幅讲的非必要语法

> I believe it is good / Let's destroy it.
> —Porno for Pyros, "Porno for Pyros"

C 可能是相对简单的语言，但 C 标准约 700 页，除非你想毕生研究它，否则重要的是知道哪些部分可以忽略。

从二合字母（digraph）和三合字母（trigraph）开始。若键盘缺 `{` 和 `}`，可用 `<%` 和 `%>` 替代。三合字母 `??<` 和 `??>` 如此无用，gcc 和 clang 的作者都没实现解析它们的代码。

## 不必显式从 main 返回

程序必须有 main 函数，返回类型必须是 int。但 C 标准规定「到达终止 main 函数的 `}` 即返回值 0」。所以不必写 `return 0;`。

```c
#include <stdio.h>
int main(){ printf("Hello, world.\n"); }
```

## 让声明流动

C89 要求所有声明在块开头，因早期编译器技术限制。C99 起可在首次使用时声明：

```c
int main(){
    double denom = 7;
    char *head = "There is a cycle to things divided by seven.";
    printf("%s\n", head);
    for (int i=1; i<= 6; i++){
        double ratio = i/denom;
        printf("%g\n", ratio);
    }
}
```

## 少用类型转换

C 在类型转换上较宽松，许多转换隐式进行。C++ 更依赖类型，需要显式转换。C 中仅在两种情况下需要显式转换：降级（如 double 到 int）和消除 void 指针的歧义。

## 枚举与字符串

用单字符作为枚举值通常足够助记，如 `'t'` 表示转置，`'p'` 表示路径错误。

## goto

goto 在错误处理中很有用，可跳到清理代码块。教科书过度妖魔化了它。

## switch

POSIX 的 getopt 等函数常用 switch。记得每个 case 后加 break，或用 fall-through 有意为之。

## 整数类型

做严肃计数时，用 `#include <stdint.h>` 和 `intmax_t` 类型，保证至少到 2^63-1。

## 标记特殊数值的 NaN

用 `NAN` 和 `isnan()` 标记非数字值，便于在数组中作哨兵。
