# 第6章 B-Tree 变体

B-Tree 变体具有共同点：树结构、通过分裂与合并的平衡、查找与删除算法。并发、磁盘页表示、兄弟节点链接、维护过程等细节因实现而异。

本章讨论若干实现高效 B-Tree 的技术及采用它们的结构：

- **写时复制 B-Tree**：结构类似 B-Tree，但节点不可变、不就地更新，页被复制、修改并写入新位置
- **惰性 B-Tree**：通过缓冲节点更新减少后续同节点写的 I/O
- **FD-Tree**：以类似 LSM 的方式缓冲更新，小 B-Tree 满时写入不可变 run
- **Bw-Tree**：将节点拆成若干小部分以仅追加方式写入，用 CAS 实现无锁
- **缓存无关 B-Tree**：以与内存结构类似的方式处理磁盘结构

## 写时复制

部分数据库使用写时复制（copy-on-write）而非复杂闩锁机制来保证并发下的数据完整性。页即将修改时，复制其内容，修改副本而非原页，并创建并行树层次。旧树版本对与写者并发的读者仍可访问；访问修改页的写者需等待前序写完成。新页层次创建后，最顶层指针原子更新。

缺点：需要更多空间和处理时间。优点：读者无需同步，因写入的页不可变；写者在副本上操作，读者不阻塞写者；不会观察到不完整页，崩溃不会留下损坏页。

### 实现：LMDB

LMDB（Lightning Memory-Mapped Database）是使用写时复制的存储引擎。由于其设计，LMDB 不需要页缓存、预写日志、检查点或压缩。LMDB 作为单层数据存储实现，读写通过内存映射直接完成。更新时，从根到目标叶路径上的每个分支节点被复制并可能修改。LMDB 仅保留两个根版本：最新版和即将提交的版本。

## 抽象节点更新

更新磁盘上的页需先更新其内存表示。可选的表示方式：直接访问缓存版本、通过包装对象访问、或创建实现语言原生的内存表示。将磁盘页、缓存版本和内存表示分开管理可让它们有不同的生命周期，例如缓冲插入、更新、删除，在读取时与原始磁盘版本协调。

## 惰性 B-Tree

惰性 B-Tree 通过缓冲更新减少 B-Tree 更新成本，使用更轻量、对并发和更新友好的内存结构缓冲更新并延迟传播。

### WiredTiger

WiredTiger 是 MongoDB 的默认存储引擎。其行存储 B-Tree 对内存页和磁盘页使用不同格式。内存页持久化前需经过协调（reconciliation）过程。干净页仅由索引组成；更新先存入更新缓冲区。读取时合并更新缓冲区与原始磁盘页内容；刷新时协调并持久化。

### 惰性自适应树（LA-Tree）

LA-Tree 将节点分组为子树，为每个子树附加更新缓冲区。缓冲区有层次依赖并级联：更新从高层缓冲区传播到低层。到达叶层时，批量执行插入、更新、删除。WiredTiger 和 LA-Tree 都通过批处理写操作优化树更新时间。

## FD-Tree

FD-Tree（Flash Disk Tree）由小可变头树（head tree）和多个不可变有序 run 组成。头树填满时，其内容转移到不可变 run；若新 run 超过阈值，与下一层合并，数据逐层向下传播。

### 分数级联

FD-Tree 使用分数级联（fractional cascading）维护层间指针，降低在有序数组级联中定位项的成本：在最高层二分查找，后续层从上一层的最近匹配开始，搜索空间显著缩小。通过从低层抽取每第 N 个元素到高层建立桥（bridge）。

### 对数 Run

FD-Tree 将分数级联与对数大小的有序 run 结合：run 大小按因子 k 增长，由合并上一层与当前层创建。删除通过插入 tombstone（filter entry）实现，tombstone 传播到最底层后可丢弃。

## Bw-Tree

Bw-Tree 通过仅追加存储、将节点链接成链、使用 CAS 安装指针实现无锁，同时解决写放大、空间放大和并发复杂度问题。

### 更新链

Bw-Tree 将基节点与修改分开写入。修改（delta 节点）形成链：从最新修改经较旧修改到基节点的链表。每次更新单独存储，无需重写现有节点。节点作为逻辑实体：无需预分配、定长或连续内存。读取时需遍历并应用所有 delta 重建实际状态。

### 用 CAS 控制并发

Bw-Tree 节点有逻辑 ID，使用内存映射表从 ID 映射到磁盘位置。通过 CAS 更新映射表中的物理偏移实现无锁。更新步骤：定位目标叶、创建新 delta 节点、用 CAS 更新映射表指向新 delta。

### 结构修改操作（SMO）

分裂 SMO：先整合分裂节点逻辑内容，创建新页，追加分裂 delta 通知读者，最后更新父节点。合并 SMO：在右兄弟上追加移除 delta，在左兄弟上创建合并 delta 指向右兄弟内容，从父节点移除右兄弟链接。

### 整合与垃圾回收

Delta 链过长时需整合：将基节点与所有 delta 合并为新基节点，写入新位置并更新映射表。使用基于 epoch 的回收：在某一 epoch 中被替换的节点保留到该 epoch 或更早开始的读者全部完成后再回收。

## 缓存无关 B-Tree

缓存无关结构在不了解缓存行、文件系统块、磁盘页大小的情况下，对任意内存层次渐近最优。van Emde Boas 布局在边的中间层分割树，递归分割得到 $\sqrt{N}$ 大小的子树，任意递归树存储在连续内存块中。动态化使用 packed array 结构，元素间预留间隙。截至写作时，尚无非学术的缓存无关 B-Tree 实现。

## 小结

原始 B-Tree 在 SSD 上存在写放大和空间开销问题。惰性 B-Tree（如 WiredTiger、LA-Tree）通过缓冲减少 I/O。FD-Tree 通过不可变性减少空间放大。Bw-Tree 通过不可变性和无锁设计同时改善写放大、空间放大和并发。Bw-Tree 的修改版已用于 Sled 存储引擎。
