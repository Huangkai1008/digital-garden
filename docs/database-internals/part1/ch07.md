# 第7章 日志结构存储

> 会计师不用橡皮擦，否则会进监狱。——Pat Helland

会计师修改记录时，不擦除原值，而是创建带修正的新记录。不可变存储结构同样不允许修改现有文件：表写入一次后不再修改，新记录追加到新文件，要得到最终值需从多个文件重建。

与可变结构（如 B-Tree）就地更新页不同，不可变结构（如 LSM Tree）使用仅追加存储和合并协调。就地更新结构优化读性能；仅追加存储优化写性能，但读需检索多版本并协调。

## LSM Tree

LSM Tree（Log-Structured Merge Tree）使用缓冲和仅追加存储实现顺序写。LSM Tree 写入不可变文件并随时间合并，这些文件通常自带索引。LSM Tree 延迟数据文件写入，在内存表中缓冲变更，然后通过刷新传播到不可变磁盘文件。

保持数据文件不可变有利于顺序写：数据单次写入，文件仅追加。插入、更新、删除无需在磁盘上定位记录，显著提高写性能和吞吐。读写设计上不交叉，无需段锁，简化并发访问。

### LSM Tree 结构

LSM Tree 由较小的内存驻留组件和较大的磁盘驻留组件组成。内存组件（memtable）可变，缓冲数据记录；磁盘组件由刷新内存内容创建，仅用于读。

**双组件 LSM Tree**：仅有一个磁盘组件，组织为 B-Tree，100% 节点占用。刷新时，内存子树与对应磁盘子树合并写入新段。

**多组件 LSM Tree**：整个 memtable 内容单次刷新。多次刷新后会有多个磁盘表，需定期压缩（compaction）合并表以控制数量。

### 更新与删除

LSM Tree 中插入、更新、删除无需在磁盘上定位记录。删除需显式记录：插入 tombstone（删除标记）表示移除某键的关联记录。协调过程会过滤被 tombstone 覆盖的值。可使用谓词删除（range tombstone）删除键范围。

### LSM Tree 查找

查找需访问多个组件，合并协调后才能返回。使用多路归并排序：用最小堆等优先队列维护各迭代器头部，每次取最小元素，从对应迭代器补充下一个元素，直到满足查询条件或迭代器耗尽。同键多记录需协调，通常按时间戳取最新。

### 维护：压缩

压缩选取多个磁盘表，用归并和协调算法迭代全部内容，将结果写入新表。Tombstone 在压缩时不能立即丢弃，需保留到能确定其他表中无更早时间戳的同键记录。RocksDB 保留到最底层；Cassandra 保留到 GC grace period。

**分层压缩（Leveled compaction）**：按层组织表，每层有目标大小。L0 表由 memtable 刷新创建，键范围可重叠。L1 及更高层的表键范围不重叠。层间大小指数增长。

**按大小分层压缩（Size-tiered compaction）**：按大小分组表，小表与小表合并，大表与大表合并。可能发生表饥饿（table starvation）问题。

## 读、写与空间放大

不可变存储面临三个问题：

| 类型 | 说明 |
|------|------|
| **读放大** | 需访问多表才能检索数据 |
| **写放大** | 压缩过程持续重写导致 |
| **空间放大** | 存储同键的多个记录 |

RUM 猜想：减少其中两个开销必然导致第三个变差。B-Tree 优化读；LSM Tree 不要求在写时定位记录，不预留额外空间，但默认配置下读更昂贵。

## 实现细节

### 有序字符串表（SSTable）

磁盘表常实现为 SSTable：数据记录按键序排列。SSTable 通常包含索引文件和数据文件。索引支持对数或常数时间查找。数据文件为拼接的键值对。压缩时可顺序读取数据文件而无需访问索引。

### Bloom 过滤器

Bloom 过滤器是空间高效的概率数据结构，用于测试元素是否在集合中。可能产生假阳性，不会产生假阴性。若返回阴性则可跳过该文件。使用位数组和多个哈希函数，通过配置位集大小和哈希函数数量控制假阳性概率。

### 跳表（Skiplist）

跳表是保持有序数据的内存结构，实现简单，概率复杂度接近搜索树。不需要旋转或重定位，使用概率平衡。由不同高度的节点构成链接层次，允许跳过元素范围。查找从最高层开始，遇到大于目标键的节点时回退到下一层前驱。

### 磁盘访问

LSM Tree 实现需考虑预取、批量读取、压缩等以优化磁盘访问。

### 无序 LSM 存储

Bitcask、WiscKey 等允许按插入顺序存储，在写路径上有优势。

### LSM Tree 中的并发

读写设计上不交叉。memtable 切换、压缩等需适当同步。

### 日志堆叠

闪存转换层（FTL）、文件系统日志等与 LSM 的日志结构理念相关。LLAMA 和 Mindful Stacking 将日志结构思想应用于开放通道 SSD 等场景。

## 小结

不可变存储结构不修改现有文件，新记录追加到新文件。LSM Tree 通过缓冲和仅追加存储实现顺序写，适合写多读少的场景。查找需合并协调多组件。压缩减少表数量、回收空间、传播 tombstone。RUM 猜想描述读、写、空间放大之间的权衡。SSTable、Bloom 过滤器、跳表等是实现 LSM Tree 的关键组件。
